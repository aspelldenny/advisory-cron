### Edge cases / limitations phát hiện thêm:

- **`NoopLaunchctl` dead_code warning:** `NoopLaunchctl` is `pub` (forward-declared for future lib-target use) but never constructed in the binary compilation. `cargo clippy --all-targets -- -D warnings` flags it. Fix: `#[allow(dead_code)]` annotation on the struct. This is a legitimate forward-declaration suppression, not a production smell.
- **`_config` field in `unregister::Args`:** clap strips leading underscore from field name, so `_config: Option<PathBuf>` renders as `--config` in `--help`. This is correct (symmetry with `register --config`). Field named `_config` in Rust to suppress unused-variable warning without `#[allow(dead_code)]` on a field that IS accessible (different from struct-level).
- **Integration test real launchctl pollution:** `tests/cli_register.rs` spawns the compiled binary which uses `RealLaunchctl`. On macOS dev machine, `register` successfully bootstraps the test label into `gui/$UID`. Mitigation: unique PID-based label suffix + best-effort `launchctl bootout` + file cleanup per test. Observed: tests pass cleanly on dev machine.
- **`launchctl bootstrap` with TempDir plist:** When HOME is overridden via `env("HOME", temp_path)`, the plist path becomes `<tempdir>/Library/LaunchAgents/com.advisorycron.<label>.plist`. `launchctl bootstrap` is called with this temp path. launchd accepts it and loads the job. The cleanup `launchctl bootout gui/$UID/com.advisorycron.<label>` in the test removes it from the live session.
- **Binary size post-P003:** `target/release/advisory-cron` = ~3.8MB (confirmed `ls -lh`). Well within 7MB budget.
- **`cargo update --dry-run` finding:** one minor dep bump available (`displaydoc 0.2.5 → 0.2.6`). Not a major bump; no action needed per INV-5.
