### Edge cases / limitations phát hiện thêm

- **`let _ = for (i, line) in reader.lines().enumerate()`** — the line index in `read_last_n` is used only for error messages. If a file has 1000+ records, all are read into memory before taking the last `n`. Phase 1 acceptable (heartbeat.jsonl is a diagnostic file — fires ≤1/day, unlikely to grow past a few MB). Phase 2 should add tail-file reading (read from end) if scaling is needed. Logged as known limitation in `heartbeat.rs` module doc.
- **`tail_utf8` snap direction:** phiếu spec says "walk forward to next char boundary." Worker confirmed: when naive cut lands mid-byte of a multi-byte char, `is_char_boundary` snaps forward to the start of the next char, which may yield a slightly shorter string than requested. This is correct behavior per spec — just clarifying that "snap forward" means the result may be shorter than `max_bytes`, not longer.
- **`cargo update --dry-run` showed `displaydoc v0.2.5 → v0.2.6`** — minor version bump in a transitive dep. Not a concern (no MAJOR version change). Logged per Sub-mechanism E check.
- **`PartialEq` derive added to `TaskConfig`** — phiếu spec says add `PartialEq` to `HeartbeatRecord` (for test assertions). The phiếu also implicitly requires `PartialEq` on `TaskConfig` if runner tests clone and mutate (which they do: `let mut bogus = config.clone()`). Worker added `#[derive(..., PartialEq)]` to `TaskConfig` — Tầng 2 stylistic call (doesn't change behavior, enables test ergonomics, no callers break).
