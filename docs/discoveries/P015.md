### Assumptions trong phiếu — SAI so với code thật:

- **Anchor #8 (binary path trong crontab entry):** Phiếu đề xuất README viết `~/.cargo/bin/advisory-cron` là path trong crontab entry. Thực tế: `advisory-cron register` dùng `std::env::current_exe()` để lấy path của binary đang chạy — crontab entry sẽ chứa path của binary nào được gọi lúc `register`, không phải hardcode `~/.cargo/bin/`. Trong smoke test trên WSL2 (dùng `target/release/advisory-cron`), crontab entry là `/home/sep/advisory-cron/target/release/advisory-cron run # advisory-cron: p015-smoke`. Cargo install lên `~/.cargo/bin/` thì sẽ cho path `~/.cargo/bin/advisory-cron`. README Linux quick-start step 3 đã thêm note giải thích hành vi này: "the binary path in the crontab entry mirrors the path of the binary that ran `register`." Không cần sửa thêm docs.
- `ls ~/.cargo/bin/advisory-cron` trả về "not found" — binary chưa được `cargo install`, chỉ có `target/release/advisory-cron`. Anchor #8 verified: path behavior đúng theo `current_exe()` design, không phải hardcode `~/.cargo/bin/`.
