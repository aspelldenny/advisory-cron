### Edge cases / limitations phát hiện thêm:

- **wiremock 0.6 với MockServer::expect(0):** wiremock 0.6 verify mock expectations on drop — nếu mock setup `.expect(0)` nhưng thực ra có request đến, test sẽ panic khi MockServer drop. Đây là cơ chế đúng cho "no POST" tests.

- **`false` command exit code trên macOS:** `/usr/bin/false` (hoặc `/bin/false`) luôn exit 1 trên macOS. Exit code từ `advisory-cron run` = 4 (task fire failed). Integration test kiểm tra `exit_code=1` trong heartbeat JSONL. Confirmed hoạt động đúng.

- **`true` command:** `/usr/bin/true` exit 0. Integration test `run_successful_task_sends_no_post` xác nhận exit 0 và 0 POST to Telegram.

- **Config struct `alert` field và TOML serialization:** `#[serde(default)]` đảm bảo `advisory-cron init` tạo config file không có `[alert]` block (alert = None → toml không serialize optional None fields trong TOML). Verified qua `write_default_creates_parent_dirs_and_file` test — round-trip vẫn load clean.

- **Binary size delta:** 2.1MB (P006/P007) → 3.9MB (P008). Delta ~1.8MB từ wiremock + h2 + deadpool deps (dev-dep ảnh hưởng release binary? Không nên). Nhưng thực tế wiremock là dev-dep. Binary bloat 1.8MB có thể từ reqwest h2 support (mới enabled?). Vẫn trong budget 7MB. Không gây ra concern ngay — ghi nhận để theo dõi.
