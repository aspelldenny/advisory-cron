### Edge cases / limitations phát hiện thêm:

- **`heartbeat_appended: bool` Tầng 2 decision:** Worker giữ `bool` = semantics của last attempt. Lý do: (a) không có consumer nào (`cli/run.rs`, `mcp/tools.rs`) display field này theo cách gây nhầm lẫn; (b) thay thành `u32` count sẽ cần update `RunOutput` consumers và integration tests — scope creep; (c) comment trong `RunOutput` struct giải thích rõ "last-attempt semantics". Đã thêm docstring giải thích rõ ràng. Kết quả: `bool` giữ nguyên, comment clarify.

- **`tokio::time::sleep` zero duration:** Khi `backoff_secs = 0`, `sleep(Duration::from_secs(0))` vẫn await nhưng yield ngay lập tức. Tests dùng `backoff_secs=0` verify logic mà không cần wait thật. Behavior hợp lý.

- **Config struct field ordering:** Thêm `retry: Option<RetryConfig>` sau `alert: Option<AlertConfig>` trong `Config` struct. Vì TOML deserialization dùng serde, field ordering không ảnh hưởng backwards-compat — chỉ cần `#[serde(default)]`.

- **Existing test Config initializers:** 2 test helpers (`src/launchd.rs` + `src/runner.rs`) cần thêm `retry: None` để compile. `src/config.rs` test helpers dùng TOML string deserialization (không affected). Structural necessity — không phải logic change.

- **`max_attempts.max(1)` defense-in-depth:** Được implement đúng. Kết hợp với `Config::validate()` bắt `max_attempts < 1`, có 2 lớp bảo vệ chống zero-attempts edge case.

- **Clippy `too_many_arguments`:** `write_retry_config` helper trong integration test có 8 arguments → clippy cảnh báo. Fix bằng `#[allow(clippy::too_many_arguments)]` — đây là test helper, không phải production code. Tầng 2 self-decide.
