## Discovery Report — P006

### Assumptions trong phiếu — ĐÚNG:

- `rmcp 1.7.0` available on crates.io with `server` + `transport-io` features. `ServerHandler` trait has `get_info`, `list_tools`, `call_tool` methods. `Tool::new(name, desc, schema)` takes `Into<Arc<JsonObject>>`. `CallToolResult::success/error` take `Vec<Content>`. `Content::text(s)` creates text content. All confirmed via `~/.cargo/registry/src/` source probe.
- `stdio()` from rmcp's transport module returns `(tokio::io::Stdin, tokio::io::Stdout)` and is compatible with `current_thread` runtime.
- `ServiceExt::serve()` returns a `RunningService` with `.waiting()` that blocks until stdin EOF — correct exit behavior for MCP stdio server.
- Hand-written JSON schemas via `serde_json::json!` + `Arc<Map>` work cleanly as Tool input schemas (Decision 3 confirmed — no `schemars` dep needed).
- All `cli::*` handlers could be rewritten as thin shells over `core::*` without changing external behavior.
- 94 tests achievable: 65 unit + 29 integration (70 baseline preserved + 24 new).
- Binary size stayed well under 7MB budget: 2.1MB release build.

### Assumptions trong phiếu — SAI so với code thật:

- **`ServerInfo` và `Implementation` là non-exhaustive structs** — phiếu assumed struct literal construction `ServerInfo { server_info: ... }`. Thực tế: both are `#[non_exhaustive]`. Fix: `let mut info = ServerInfo::default(); info.server_info = Implementation::new(name, version); info`. `Implementation::new(name, version)` is the correct constructor.
- **`ListToolsResult` cần field `meta: None`** — phiếu didn't note it. Non-exhaustive struct requires all known fields. Fix: added `meta: None`.
- **Clippy `default_constructed_unit_structs`** — `AdvisoryCronHandler::default()` on a unit struct triggers clippy lint. Fix: use bare `AdvisoryCronHandler` (no `::default()`).
- **Clippy `manual_async_fn`** — RPIT return type on `list_tools` and `call_tool` trait impl methods triggers lint. Fix: `async fn` syntax in impl.
- **`rmcp::ErrorData` path** — phiếu referenced `rmcp::error::ErrorData`. Actual re-export path: `rmcp::ErrorData` (mod `error` is private). Fix: `use rmcp::ErrorData`.
- **`tests/cli_mcp.rs` cannot use library internals** — no `lib.rs` exists; integration tests can only access the binary. Fix: rewrote all 7 tests as binary subprocess tests using `Command::new(BIN)` + JSON-RPC message exchange via piped stdio.
- **`parse_next_fire` trong `cli/status.rs`** — phiếu spec said move to `core/status.rs`. Confirmed done. ARCHITECTURE.md `src/launchd.rs` row note updated (no longer mentions `parse_next_fire` as being in `cli/status.rs`).

### Edge cases / limitations phát hiện thêm:

- **Backward compat warning messages for unregister** — existing test `unregister_nonexistent_label_exits_0_idempotent` asserted "warning" text in stderr. After extracting to `core::unregister`, the CLI shell had no warnings. Added explicit `eprintln!("warning: launchctl bootout: ...")` when `!output.was_loaded` and `eprintln!("warning: plist file already absent...")` when `!output.plist_existed` in `cli/unregister.rs`. These are Tầng 2 (user-visible warning wording on CLI, not MCP surface). If wording changes needed in future — Tầng 1 for CLI, free for MCP (MCP returns structured JSON).
- **MCP `tools/call` for unknown tool name returns `is_error=true`** — not an exception/panic. `dispatch_tool` has a catch-all arm returning `tool_error(format!("unknown tool: {other:?}"))`. Consistent with MCP protocol behavior (tool errors as results, not JSON-RPC errors).
- **MCP server exits cleanly on stdin EOF** — `service.waiting().await` returns `Ok(())` when rmcp reads EOF. Binary exits 0. Claude Desktop disconnecting gracefully will not leave zombie processes.
- **`tokio io-std` feature** — without this tokio feature, rmcp's stdio transport cannot compile. The feature was confirmed necessary via compile error on first attempt without it.
- **`cargo test --all` runs binary integration tests sequentially** — MCP tests spawn the binary subprocess with `HOME` set to a TempDir. No port conflicts (stdio only). Tests are hermetic and fast (~1s total for 7 MCP tests).
